### Contributing

Thanks for your interest in improving the Completions → Responses migration pack.

How to contribute:
- Open an issue describing the problem or proposed enhancement
- Fork, create a feature branch, and submit a PR with a clear description
- Keep edits focused and small; include before/after notes when changing behavior

Coding guidelines:
- Prefer clear, defensive shell code; keep the bootstrap script POSIX-friendly
- Avoid leaving temporary files or comments (“tombstones”) in user repos
- Do not write outside the target git workspace
- Follow the docs structure (`docs/*`) and update the cheat‑sheet or recipes when changing behavior

Security & secrets:
- Never log secrets; do not commit `.env` files
- If you discover a security issue, please follow `SECURITY.md`
